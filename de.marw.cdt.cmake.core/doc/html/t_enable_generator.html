Enabling CMake buildscript generation

To have your buildscripts generated by cmake, open the Project➢ Properties dialog.
Expand the C/C++ Build node and click onto Tool Chain Editor.
On the tool chain editor page, select either CMake Make Builder or CMake Builder (portable) as the current builder.

CMake Make Builder

This builder is nearly identical to the built-in Gnu Make Builder, except that it uses the information found in the CMakeLists.txt file to generate the buildscripts. It will always generate makefiles and then invoke the make tool found in the system's executable search path.

CMake Builder (portable)

This builder allows to customize the build tool to be used on the OS-specific property tabs and makes CMake generate the build scripts for the tool of choice. In contrast to the CMake Make Builder, which is restricted to the make tool, it allows to build with make, ninja, nmake and others.
Since CMake does a fairly good job in detecting the absolute file system location of the build tool, the latter can be invoked even if it is not on the executable search path.
