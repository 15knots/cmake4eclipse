Project Builders

The CMake for CDT plugin provides the following Project Builders (selectable in the Tool Chain Editor tab):

CMake Builder (portable)

This builder allows you to customize the build tool to be used on the OS-specific property tabs and makes CMake generate the build scripts for the tool of your choice.

The builder retrieves the name of the actual build tool from the cache file written by cmake during build-script-generation. It has some benefits compared to the CMake Make Builder:

  • It allows you to build with tools other than make, for example ninja, nmake and others.
  • You are not forced to put the build tool into $PATH on your machine, since cmake does a fairly good job to detect the absolute file system path of the build tool.
  • It makes your Eclipse project being a cross-platform project: You can configure it to build using make under Linux, and build with MSYS make under windows; without the need to change platform-specifc project settings after checkout.
This is the recommended builder.

CMake Make Builder

This builder is nearly identical to the built-in Gnu Make Builder, except that it uses the information found in the CMakeLists.txt file to generate the buildscripts. It will always generate makefiles and then simply invoke the make tool found in the system's executable search path ($PATH) to build the project.

Please note that this builder originates from the original states of this plugin and is kept for compatibility reasons only.
