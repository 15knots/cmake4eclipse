Project Builders

The CMake for CDT plugin provides the following Project Builders. (selectable in the Tool Chain Editor tab):

CMake Builder (portable)

This builder retrieves the name of the build tool from the cache file written by cmake during build-script-generation. This approach has some benefits:

  • It allows you to build with tools other than make, for example ninja.
  • Since cmake does a fairly good job to detect the absolute file system path of the build tool, you are not forced to put the tool into $PATH.
  • It makes your Eclipse project being a cross-platform project: Build using make under Linux, build with MSYS make under windows; without adjusting the Eclipse project files.
This is the recommended builder.

CMake Make Builder

This builder simply invokes the make command from the executable search path ($PATH) to build the project. This builder originates from the original states of this plugin and is kept for compatibility reasons only.
