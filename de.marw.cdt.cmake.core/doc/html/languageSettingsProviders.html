Preprocessor Include Paths, Macros, etc.

Include paths and preprocessor macros for the C/C++ indexer are supplied by Language Settings Providers using Scanner Discovery mechanism.

The CMake for CDT plugin provides these Settings Providers:

CMAKE_EXPORT_COMPILE_COMMANDS Parser

This provider instructs CMake to generate a file that contains the compiler commands-lines which will be executed when a project is build. It parses that file and provides the preprocessor include paths, macros definitions and macros undefines to the indexer. Parsing is done at most once per build since the cache is invalidated when the file`s time-stamp is newer, so it should be reasonably fast.

CMake Build Output Parser (deprecated)

Do not use, it will be removed in future versions. It is just my first implementation of aLanguageSettingsProvider.
This provider parses each line of the build output and tries to find compiler command-lines. To do so, it instructs CMake to generate build scripts that produce verbose build output. As a cause, it will not work with the ninja build tool, because ninja builds do not print any compiler command lines by default.
Build output parsing degrades build performance, so this provider tries to cache its findings -- unfortunately its caching is somehow unreliable.
