Getting Started

Cmake4eclipse requires an existing CDT C-project or C++-project to work with.
It allows to use cmake as the generator for the buildscripts instead of the makefile generator built-in to CDT.

As a first start, you may want to try out one of the example projects.

To set up your own project with existing source code, please follow these steps:

  1. Check out your source code.
  2. Open menu File➢ New➢ Project...➢ C/C++ and select one of the New C project or New C++ project wizards. Do not choose the New C/C++ project wizard, as that will take you down the road of CDT`s new project type (which has poor cmake support).
  3. Make sure the project location points to root directory of your checked out files
  4. For the project type, select anything below Executable, Shared Library or Static Library, it does not matter since the settings actually come from your CMakeLists.txt files.
    Do not select Makefile project here!
  5. Finish project creation.
  6. Open the Project Properties dialog.
  7. Select the C/C++ Build node and the Builder Settings tab. Make sure that
    1. External Builder is set as the builder type.
    2. Use default build command is checked. (Although you may use different settings here at your own risk.)
    3. Generate Makefiles automatically is checked.
  8. Select the Tool Chain Editor node and set CMake Builder (portable) as the current builder. See Builders for details on the available builders.
  9. If your top level CMakeLists.txt does not reside in the root directory of your checked out files, select the C/C++ General➢ Path and Symbols node and the Source Location tab. Then adjust the source folder to point to the directory containing your top level CMakeLists.txt file. This will tell the CDT indexer to scan the header files in your project.
    Pro Tip: Add a CMakeLists.txt file in the root directory of your checked out files, if you miss the Binaries or Archives folder in the C/C++ Projects View.
  10. Build the project. This will invoke cmake to generate the build scripts, if necessary, and then will invoke make.

Do not try to import an Eclipse project that you created manually using cmake -G Eclipse CDT4 - Unix Makefiles, as that will put you on the classic Makefile project route!
