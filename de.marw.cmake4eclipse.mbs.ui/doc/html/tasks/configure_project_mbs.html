Configuring an existing Eclipse C/C++-project

Cmake4eclipse requires an existing CDT C/C++ project to work with.
It does not work with Eclipse projects generated by the cmake -G Eclipse CDT4 - Unix Makefiles command.

To configure an existing Eclipse C/C++ project, check the following project properties:

  1. Open the Project Properties dialog.
  2. Select the C/C++ General ➢ Preprocessor Include Path. Macros etc node and the Providers tab and make sure the CMake Compilation DB provider is checked. You may safely uncheck any other provider here.
  3. Build the project. This will first invoke cmake to generate the build scripts and then pass control to CDT to run the generated scripts. You will find the full build output in the CDT Global Build Console of the Console View.
