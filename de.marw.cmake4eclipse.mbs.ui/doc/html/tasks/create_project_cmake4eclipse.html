Creating a Cmake4eclipse Managed Build project

To set up your own project with or without existing source code, please follow these steps:

  1. Check out your source code.
  2. Open menu File ➢ New ➢ Project... and expand the C/C++node. Then select the C/C++ Project wizard.
  3. Click the Next button to enter the project creation wizard step.
  4. On the template selection page, select Cmake in the left panel and Cmake4eclipse Managed Build in the right panel.
  5. Click the Next button to enter the project configurations wizard step.
  6. Make sure the project location points to the root directory of your checked out files.
  7. Click the Next button to enter the project build configurations wizard step.
  8. Check the configurations you want to build for.
  9. If your top level CMakeLists.txt does not reside in the project`s root directory , click the Advanced settings... button and
    1. select the Cmake4eclipse Build node,
    2. then the Source Location tab
    3. and adjust the source folder to point to the directory containing your top level CMakeLists.txt file.
    4. Click the Apply and Close button to return to the project build configurations wizard step.
  10. Finish project creation.
  11. Build the project. This will first invoke cmake to generate the build scripts and then pass control to CDT to run the generated scripts.
    To examine the full build log, open Eclipse`s Console View, then set it to display the CDT Global Build Console. There is also a CMake Console which will display the output of the most recent cmake invocation only.
