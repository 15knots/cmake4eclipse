Creating a C or C++ Managed Build project and configuring it

This requires an existing CDT C-project or C++-project to work with.
It allows to use cmake as the generator for the buildscripts instead of the makefile generator built-in to CDT.

To set up your own project with existing source code, please follow these steps:

  1. Check out your source code.
  2. Open menu File ➢ New ➢ Project... and expand the C/C++node. Then select one of the C Project or C++ Project wizards.
  3. Click the Next button to enter the project creation wizard step.
  4. Make sure the project location points to the root directory of your checked out files.
  5. For the project type, expand the CMake driven node, select Empty Project and the CMake driven toolchain.
  6. Click the Next button to enter the project configurations wizard step.
  7. Check the configurations you want to build for.
  8. If your top level CMakeLists.txt does not reside in the project`s root directory , click the Advanced settings... button and
    1. select the C/C++ General➢ Path and Symbols node,
    2. then the Source Location tab
    3. and adjust the source folder to point to the directory containing your top level CMakeLists.txt file.
    4. Click the Apply and Close button to return to the project build configurations wizard step.
    Pro Tip: If you later miss the Binaries or Archives folder in the Project Explorer View , add a CMakeLists.txt file in the root directory of your checked out files.
  9. Finish project creation.
  10. Build the project. This will first invoke cmake to generate the build scripts and then pass control to CDT to run the generated scripts.
    To examine the full build log, open Eclipse`s Console View, then set it to display the CDT Global Build Console. There is also a CMake Console which will display the output of the most recent cmake invocation only.
