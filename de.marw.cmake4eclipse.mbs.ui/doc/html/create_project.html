Creating a Project

Cmake4eclipse requires an existing CDT C-project or C++-project to work with.
It allows to use cmake as the generator for the buildscripts instead of the makefile generator built-in to CDT.

Do not try to manually generate an Eclipse project using cmake -G Eclipse CDT4 - Unix Makefiles and import it, as that will make cmake4eclipse unavailable in your project!

As a first start, you may want to try one of the example projects.

To set up your own project with existing source code, please follow these steps:

  1. Check out your source code.
  2. Open menu File ➢ New ➢ Project... ➢ C/C++ and select one of the C Project or C++ Project wizards. Do not choose the C/C++ project wizard, as that will take you down the road of CDT`s new project type which does not recognize cmake4eclipse.
  3. Make sure the project location points to the root directory of your checked out files.
  4. For the project type, expand the CMake driven node, select Empty Project and the CMake driven toolchain.
  5. Click the Next button to enter the project configurations wizard step.
  6. Check the configurations you want to build for. The available configurations map cmake`s CMAKE_BUILD_TYPE variable as following:
    • Debug sets CMAKE_BUILD_TYPE=Debug.
    • Release sets CMAKE_BUILD_TYPE=Release.
    • Default does not set CMAKE_BUILD_TYPE at all, allowing you to use cmake`s default value.
    Please note that if you specify a value for CMAKE_BUILD_TYPE on the Symbols tab that value will override the value from the configuration.
  7. If your top level CMakeLists.txt does not reside in the project`s root directory , click the Advanced settings... button and
    1. select the C/C++ General➢ Path and Symbols node,
    2. then the Source Location tab
    3. and adjust the source folder to point to the directory containing your top level CMakeLists.txt file.
    Pro Tip: If you miss the Binaries or Archives folder in the Project Explorer View , add a CMakeLists.txt file in the root directory of your checked out files.
  8. Finish project creation.
  9. Build the project. This will first invoke cmake to generate the build scripts and then pass control to CDT to run the generated scripts.
    To examine the full build log, open Eclipse`s Console View, then set it to display the CDT Global Build Console. There is also a CMake Console which will display the output of the most recent cmake invocation only.
