Configuring an existing Eclipse C-project or C++-project

Cmake4eclipse requires an existing CDT C-project or C++-project to work with.
It does not work with projects created by the more recent C/C++ project wizard of CDT nor will it work with Eclipse projects generated by the cmake -G Eclipse CDT4 - Unix Makefiles command.

To configure an existing Eclipse C-project or C++-project, check the following project properties:

  1. Open the Project Properties dialog.
  2. Select the C/C++ Build node and the Builder Settings tab. Make sure
    1. that External Builder is set as the builder type,
    2. that Use default build command is checked. (Although you may use different settings here at your own risk.)
    3. that Generate Makefiles automatically is checked.
  3. Select the Tool Chain Editor node and set CMake Builder as the current builder.
  4. Optional: Set CMake driven as the current toolchain.
  5. Select the C/C++ General ➢ Preprocessor Include Path. Macros etc node and the Providers tab and make sure the CMake Compilation DB provider is checked. You may safely uncheck any other provider here.
  6. If your top level CMakeLists.txt does not reside in the root directory of your checked out files, select the C/C++ General ➢ Path and Symbols node and the Source Location tab. Then adjust the source folder to point to the directory containing your top level CMakeLists.txt file.
  7. Build the project. This will first invoke cmake to generate the build scripts and then pass control to CDT to run the generated scripts. You will find the full build output in the CDT Global Build Console of the Console View.
